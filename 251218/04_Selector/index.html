ID selector

  • 특정한 요소를 선택하고자 할 때 사용한다.
  • html 요소에 id를 부여한다.
  • CSS에서 요소의 id 앞에 #을 붙인다.
  • 한 문서에 특정 id는 한개여야만 한다. 중복된 이름의 id가 존재하면 안된다.
  • id명은 숫자로 시작할 수 없다.
  • 몇 개의 요소를 하나의 클래스로 묶어서 스타일을 지정하고자 할 때 사용한다.
  • html 요소에 클래스를 부여한다.
  • CSS에서 클래스명 앞에 .을 붙여 호출가능하다.
  • 클래스 선택자는 여러 개의 요소를 한꺼번에 선택할 수 있다.
    (class=B/id=name2 | font-family sans | .parent>#child CSS 적용중.)
  • 클래스명은 숫자로 시작하지 않는다.

class 선택자는 class로 붙인 이름을 가져다 쓰는 선택자입니다. '.'을 붙여 표기합니다.

전체 선택자

  • HTML 문서의 모든 요소를 한꺼번에 선택한다.
  • 별표(*) 기호를 사용하여 호출한다.
  • 보통 여백 초기화(margin, padding: 0) 설정 시 많이 활용한다.
  • 개별 선택자(id, class)보다 우선순위가 낮다.
  • 남용하면 브라우저 성능에 부담을 줄 수 있어 주의해야 한다.

자식 선택자

  • 현재 요소의 바로 아래에 나오는 선택자
  • (예시)ul>li라고 표현하면 ul에 포함된 li 요소를 말한다.

인접 선택자

  • 바로 뒤 요소 선택: A+B라고 쓰면, A 바로 다음에 오는 형제 요소 B 딱 하나만 선택한다.
  • 동일 부모 조건:
  • 순서의 중요성: 반드시 A가 B보다 먼저 나와야 하며, 중간에 다른 태그가 끼어들면 안된다.
  • 스타일 연결: 주로 제목(<h1>) 바로 뒤에 오는 본문(<p>)의 첫 번째 단락에만 특별한 여백을 줄 때 유용하다
  • 입력 폼 활용: 체크박스(<input>)가 체크되었을 때 바로 뒤의 글자(<label>) 색상을 바꾸는 등의 인터랙션에 자주 쓰인다.
